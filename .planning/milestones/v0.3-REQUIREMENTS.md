# Requirements: v0.3 Multi-Agent Gymnasium RL Environment

**Milestone:** v0.3
**Goal:** Gymnasium-compatible RL environment with multi-agent support for heterogeneous piece-type agents
**Created:** 2026-03-26

---

## Requirements

### R1: Multi-Agent Gymnasium Env Interface ✓
- [x] Class `XiangqiEnv` inherits `gym.Env`
- [x] `reset(seed=None, options=None) -> (obs, info)`
- [x] `step(action) -> (obs, reward, terminated, truncated, info)`
- [x] `observation_space: gym.spaces.Box(low=0.0, high=1.0, shape=(16, 10, 9), dtype=np.float32)`
- [x] `action_space: gym.spaces.Discrete(8100)` (from_sq * 90 + to_sq)
- [x] Compatible with `gymnasium.make("Xiangqi-v0")`

### R2: Multi-Agent Action Spaces (7 Piece Types) ✓
- [x] Per-piece-type action mask via `info["piece_masks"]` dict
- [x] `piece_masks: {piece_type: np.ndarray}` — 7 keys, each (8100,) float32
- [x] Piece types: GENERAL=0, ADVISOR=1, ELEPHANT=2, HORSE=3, CHARIOT=4, CANNON=5, SOLDIER=6
- [x] Each mask marks only legal moves for pieces of that type

### R3: AlphaZero-Style Board Planes Observation
- [x] 16 channels: 7 piece types × 2 colors + 2 auxiliary
- [x] Channels 0-6: Red pieces (General, Advisor, Elephant, Horse, Chariot, Cannon, Soldier)
- [x] Channels 7-13: Black pieces
- [x] Channel 14: Repetition count (normalized 0-3)
- [x] Channel 15: Halfmove clock (normalized 0-100)
- [x] Canonical rotation: board flipped so active player sees their General at bottom

### R4: Legal Move Masking
- [x] `get_legal_mask(player=1|-1) -> np.ndarray` — full 8100-element mask
- [x] `get_piece_legal_mask(piece_type, player) -> np.ndarray` — per-type mask
- [x] Illegal move in `step()` returns penalty reward, does NOT update state
- [x] Mask computation uses existing `XiangqiEngine` legal move generation

### R5: Basic Reward Signal
- [x] Terminal reward: +1.0 (win), -1.0 (loss), 0.0 (draw)
- [x] Material capture reward: +/-piece_value/100 per capture
- [x] Piece values: General=0, Advisor=2, Elephant=2, Horse=4, Chariot=9, Cannon=4.5, Soldier=1/2
- [x] Illegal move penalty: -2.0 (does not update state)

### R6: Terminal Detection ✓
- [x] Checkmate → side in check loses
- [x] Stalemate (kunbi) → player to move loses (WXF rule)
- [x] Repetition draw (4-fold)
- [x] 50-move rule draw
- [x] WXF perpetual check/chase detection

### R7: Self-Play End-to-End Validation
- [x] Random vs Random self-play completes 100 games without crash
- [x] Average game length 60-120 plies
- [x] Win/loss/draw distribution reasonable (~50/50 random vs random)
- [x] All legal moves verified against `XiangqiEngine.legal_moves()`

### R8: Thread Safety for Vectorized Env ✓
- [x] `gymnasium.vector.SyncVectorEnv` with n_envs > 1 works correctly
- [x] Each env instance maintains independent state

## Out of Scope (v1.0)

- Neural network training (policy/value networks)
- Per-piece-type policy heads
- Arbitration network
- Team reward shaping / cooperation bonuses
- Alpha-Beta / MCTS search
- Human vs AI gameplay integration

---

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Flat Discrete(8100) action space | Simplest; mask filters illegal moves; proven approach |
| 16-channel board planes | AlphaZero standard; 7 piece types × 2 colors + 2 auxiliary |
| Canonical rotation | Active player always sees board same orientation; halves learning burden |
| Per-type action masks in info dict | Multi-agent interface without changing gym API; compatible with SB3 |
| Basic material reward only | Keep v0.3 simple; team/cooperation rewards in v1.0 |
