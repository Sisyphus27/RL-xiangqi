# Phase 08 Plan 02: GameController Logic

## Goal

Extend GameController with game control logic: random side assignment, new game initialization, double-step undo, and updated status bar with side indicator.

**Purpose:** Implement core game control features per UI-08 and UI-09 requirements.
**Output:** Enhanced GameController with new_game(), undo(), and _human_side management.

## Source Files

- `src/xiangqi/controller/game_controller.py` - Controller to extend
- `src/xiangqi/engine/engine.py` - XiangqiEngine API
- `src/xiangqi/ui/board.py` - QXiangqiBoard for refresh_pieces()

## Implementation Steps

### Step 1: Add _human_side attribute and undo_available signal

Add to GameController class:
- Import `random` module
- Add `_human_side: int = 1` instance attribute (+1=Red, -1=Black)
- Add `undo_available = pyqtSignal(bool)` signal definition
- Initialize `_human_side` in `__init__` with random choice per D-07

### Step 2: Implement new_game() method

Add `new_game(self) -> None` method:
- Randomly assign `_human_side = random.choice([1, -1])` per D-07
- Call `self._engine.reset()` to clear board and undo stack
- Call `self._board.refresh_pieces()` to update UI
- Clear any active selection on board
- Update status bar with new side indicator per D-08
- If `_human_side == -1` (human plays Black): trigger AI move immediately per D-09
- Emit `undo_available.emit(False)` since undo stack is now empty

### Step 3: Implement undo() method

Add `undo(self) -> None` method per D-02, D-03:
- Call `self._engine.undo()` twice (double-step undo)
- Call `self._board.refresh_pieces()` to update UI
- Clear any active selection on board
- Update status bar (always returns to Red's turn per D-03)
- Check if more undos available: `undo_available.emit(can_undo)`
- Guard: only undo if not during AI turn (button should be disabled anyway)

### Step 4: Update _update_status_bar() with side indicator

Modify `_update_status_bar(self, turn: int, ai_thinking: bool)`:
- Add side text: "你执红方" if `_human_side == 1` else "你执黑方"
- Format: `f"{side_text} | {turn_text}"`
- Turn text options: "红方回合", "黑方回合", "AI 思考中..."
- Example: "你执红方 | 红方回合" or "你执黑方 | AI 思考中..."

### Step 5: Update interaction control for side assignment

Modify `_on_turn_changed(self, turn: int)`:
- Enable interaction only when `turn == self._human_side` per D-09
- Replace hardcoded `turn == 1` check with `turn == self._human_side`

Modify `_on_move_applied`:
- Update AI turn trigger: `if self._engine.turn != self._human_side` instead of `== -1`

### Step 6: Update AI thinking handlers for undo button state

Modify `_on_ai_thinking_started(self)`:
- Emit `undo_available.emit(False)` per D-04

Modify `_on_ai_thinking_finished(self)`:
- Check if undo is possible (engine has moves in history)
- Emit `undo_available.emit(can_undo)` per D-05

### Step 7: Initialize with random side on first start

In `__init__`:
- Set `_human_side = random.choice([1, -1])`
- If `_human_side == -1`: trigger AI move after initialization
- Update initial status bar with side indicator
