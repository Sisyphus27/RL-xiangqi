# Phase 08 Plan 01: Toolbar and UI

## Goal

Add a QToolBar to MainWindow containing New Game and Undo buttons with proper signal wiring and keyboard shortcuts.

**Purpose:** Provide user controls for starting new games and undoing moves.
**Output:** MainWindow with functional toolbar, button state management via signals.

## Source Files

- `src/xiangqi/ui/main.py` - MainWindow class to extend
- `src/xiangqi/controller/game_controller.py` - Controller to connect signals

## Implementation Steps

### Step 1: Add toolbar setup method to MainWindow

Add `_setup_toolbar()` method to MainWindow class:
- Create QToolBar with title "游戏控制"
- Add toolbar via `self.addToolBar(toolbar)`
- Create QPushButton instances: `_new_game_btn` ("新对局") and `_undo_btn` ("悔棋")
- Add buttons to toolbar via `toolbar.addWidget()`
- Set tooltips: "开始新对局 (Ctrl+N)" and "悔棋 (Ctrl+Z)"
- Call `_setup_toolbar()` from `__init__` after controller creation

### Step 2: Add keyboard shortcuts

Add QShortcut instances in `_setup_toolbar()`:
- `QShortcut(QKeySequence("Ctrl+N"), self)` -> connect to `_on_new_game()`
- `QShortcut(QKeySequence("Ctrl+Z"), self)` -> connect to `_on_undo()`
- Add import: `from PyQt6.QtGui import QKeySequence`
- Add import: `from PyQt6.QtWidgets import QShortcut`

### Step 3: Add button action handlers

Add handler methods to MainWindow:
- `_on_new_game()` - emits signal or calls controller method
- `_on_undo()` - emits signal or calls controller method

Per D-06 and D-01: New Game button always enabled.
Per D-04, D-05: Undo button disabled during AI thinking or when undo stack empty.

### Step 4: Add controller signal connections

In MainWindow, add method to connect controller signals:
- `controller.undo_available.connect(self._undo_btn.setEnabled)`
- Expose button references via properties if needed for external connection

## Verification

Run the application and verify:
1. Toolbar appears at top of window with two buttons
2. Buttons display "新对局" and "悔棋" labels
3. Ctrl+N triggers new game action
4. Ctrl+Z triggers undo action
5. Button tooltips show on hover
