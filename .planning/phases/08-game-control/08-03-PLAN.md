# Phase 08 Gap Closure Plan 03: Fix Undo Board Sync

## Objective

Fix AttributeError in `undo()` method by replacing non-existent `reset_to_state()` with correct `sync_state()` method.

**Purpose:** UAT Test 3 failed with `AttributeError: 'QXiangqiBoard' object has no attribute 'reset_to_state'`
**Output:** Fixed board sync call in undo() method

## Source Files

- `src/xiangqi/controller/game_controller.py` — line 290 has the bug
- `src/xiangqi/ui/board.py` — QXiangqiBoard.sync_state() exists at line 437

## Bug Analysis

**Problem:** Line 290 calls `self._board.reset_to_state(self._engine.state)` but QXiangqiBoard has no `reset_to_state` method.

**Available methods on QXiangqiBoard:**
- `refresh_pieces()` (line 417) — refreshes from internal `_state` (stale after engine.undo())
- `sync_state(state)` (line 437) — syncs board to engine state AND refreshes pieces

**Correct fix:** Call `sync_state(self._engine.state)` which:
1. Updates board's internal `_state` to match engine state
2. Calls `refresh_pieces()` to update visual display
