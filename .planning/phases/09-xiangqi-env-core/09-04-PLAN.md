## Gap Context (from 09-UAT.md)

**Test 1 FAILED (BLOCKER):** `gymnasium.make("Xiangqi-v0")` fails with `NameNotFound: Environment 'Xiangqi' doesn't exist.`

**Root Cause Analysis:**
1. The registration code DOES exist in `src/xiangqi/rl/env.py` at line 242
2. However, this only registers the environment when the module is **imported**
3. For `gymnasium.make("Xiangqi-v0")` to work without prior import, gymnasium needs entry point registration

## Current Implementation

From `src/xiangqi/rl/env.py` line 241-242:
```python
# Gymnasium registration (D-01, enables gymnasium.make("Xiangqi-v0"))
gym.register(id="Xiangqi-v0", entry_point="xiangqi.rl.env:XiangqiEnv")
```

This works ONLY if `import xiangqi.rl` or `import xiangqi.rl.env` runs first.

## Standard Solution

Gymnasium supports environment discovery via Python entry points. Add to `pyproject.toml`:

```toml
[project.entry-points."gymnasium.envs"]
Xiangqi = "xiangqi.rl.env:XiangqiEnv"
```

This makes the environment discoverable when gymnasium is imported.
