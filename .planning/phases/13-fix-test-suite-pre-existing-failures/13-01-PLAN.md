Per D-02: Wrap `_on_move_applied()` calls with `patch.object(controller, '_start_ai_turn')` in 2 tests to prevent real QThread creation. The `turn_changed` signal is emitted BEFORE `_start_ai_turn` is called (line 83 of game_controller.py), so mocking `_start_ai_turn` does NOT affect `turn_changed` assertions.

Note: `patch` is already imported at the top of the file (line 3: `from unittest.mock import Mock, patch`). No new import needed.

**Test 1: `test_controller_emits_turn_changed_on_user_move` (class TestGameControllerTurnChanged, starts at line 44)**

Replace lines 70-76 (the move application and handler call):
```python
        # Apply a move to the engine (simulating board move)
        move = engine.legal_moves()[0]
        engine.apply(move)

        # Trigger controller's _on_move_applied handler
        from_sq = move & 0x1FF
        to_sq = (move >> 9) & 0x7F
        controller._on_move_applied(from_sq, to_sq, 0)
```

With:
```python
        # Apply a move to the engine (simulating board move)
        move = engine.legal_moves()[0]
        engine.apply(move)

        # Trigger controller's _on_move_applied handler
        # Mock _start_ai_turn to prevent real QThread spawning (D-02)
        from_sq = move & 0x1FF
        to_sq = (move >> 9) & 0x7F
        with patch.object(controller, '_start_ai_turn'):
            controller._on_move_applied(from_sq, to_sq, 0)
```

This test checks `turn_changed` signal (emitted at line 83 of game_controller.py, before `_start_ai_turn`). Mocking `_start_ai_turn` does not affect this assertion. The existing assertions on lines 79-80 remain unchanged.

**Test 2: `test_controller_starts_ai_on_black_turn` (class TestGameControllerAIThinking, starts at line 86)**

Replace lines 107-121 (signal tracking, move application, handler call, and assertion):
```python
        # Track ai_thinking_started signal
        thinking_started = []
        controller.ai_thinking_started.connect(lambda: thinking_started.append(True))

        # Apply a red move (making it black's turn)
        move = engine.legal_moves()[0]
        engine.apply(move)

        # Trigger controller's _on_move_applied handler
        from_sq = move & 0x1FF
        to_sq = (move >> 9) & 0x7F
        controller._on_move_applied(from_sq, to_sq, 0)

        # ai_thinking_started should have been emitted
        assert len(thinking_started) == 1
```

With:
```python
        # Apply a red move (making it black's turn)
        move = engine.legal_moves()[0]
        engine.apply(move)

        # Trigger controller's _on_move_applied handler
        # Mock _start_ai_turn to prevent real QThread spawning (D-02)
        # Verify _start_ai_turn is called (proving AI turn detection works)
        from_sq = move & 0x1FF
        to_sq = (move >> 9) & 0x7F
        with patch.object(controller, '_start_ai_turn') as mock_start:
            controller._on_move_applied(from_sq, to_sq, 0)
            mock_start.assert_called_once()
```

Rationale: `_start_ai_turn` emits `ai_thinking_started` internally (line 186 of game_controller.py). Since we mock `_start_ai_turn`, that signal will not fire. Instead, we verify the method was called (proving the controller correctly detects it's AI's turn and attempts to start AI computation). This tests the same behavior -- "on black turn, AI thinking is initiated" -- without the thread race.
