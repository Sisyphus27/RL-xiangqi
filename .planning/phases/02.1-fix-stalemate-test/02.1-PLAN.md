def get_game_result(state) -> str:
    legal = generate_legal_moves(state)
    if len(legal) > 0:
        return 'IN_PROGRESS'
    in_check = is_in_check(state, state.turn)
    if in_check:
        return 'BLACK_WINS' if state.turn == +1 else 'RED_WINS'
    else:
        return 'BLACK_WINS' if state.turn == +1 else 'RED_WINS'
```

    def test_stalemate_also_loss(self):
        """Checkmate: red king has no legal moves and is in check.

        B_CHE at (9,0) attacks row 9 horizontally — king at (9,4) cannot move
        horizontally to (9,3) or (9,5) (both are on the attacked row).
        B_ADVISOR at (8,4) blocks the forward (8,4) square.
        R_SHI at (9,3) and (9,5) block the diagonal escape squares (8,3) and (8,5)
        respectively (king cannot capture own pieces).
        All 5 adjacent king squares are covered or occupied — zero legal moves.
        King is IN CHECK from B_CHE — checkmate — BLACK_WINS.

        NOTE: This is the correct xiangqi rule per RULE-06. The old position
        (R_SHI at (9,3)/(9,5) + R_BING at (8,4)) was broken because correct
        SHI movement allows R_SHI at (9,3) to move diagonally to (8,4).
        """
        state = make_state(+1, rc_to_sq(9, 4), rc_to_sq(0, 3),
                           extra_pieces={
                               (9, 0): Piece.B_CHE,     # checking chariot: attacks row 9
                               (8, 4): Piece.B_ADVISOR, # blocks forward (8,4) escape
                               (9, 3): Piece.R_SHI,    # blocks diagonal (8,3) escape
                               (9, 5): Piece.R_SHI,    # blocks diagonal (8,5) escape
                           })
        result = get_game_result(state)
        assert result == 'BLACK_WINS'
    ```

    The key insight of this design:
    - B_CHE at (9,0) attacks all of row 9, including (9,4) where the king sits and (9,3)/(9,5) where the king might escape horizontally.
    - B_ADVISOR at (8,4) blocks the forward escape. A red piece on (8,4) would be captured by the advisor, so the king cannot move there (cannot capture own piece).
    - R_SHI at (9,3) and (9,5) block the diagonal escapes. A king cannot capture its own piece.
    - R_SHI pieces have no legal moves: from (9,3) diagonals are (8,2)[OOB col], (8,4)[occupied by B_ADVISOR], (7,4)[OOB row].
    - From (9,5): (8,4)[occupied], (8,6)[OOB col], (7,4)[OOB row].
