    Fix the TypeError in `drawBackground()` at line 132 where `fillRect()` is called with float arguments.

    **Current code (lines 131-135):**
    ```python
    # 1. Green felt background rectangle (slightly larger than grid)
    p.fillRect(
        -0.6 * cell, -0.6 * cell, 10.2 * cell, 11.2 * cell,
        QBrush(QColor(BOARD_BG)),
    )
    ```

    **Problem:** PyQt6's `fillRect(x, y, w, h, brush)` overload expects integer arguments. When `cell` is a float (e.g., 53.57), all four coordinate arguments become floats, causing a TypeError because no matching overload exists.

    **Fix:** Wrap the coordinates in `QRectF` which accepts floating-point values:
    ```python
    # 1. Green felt background rectangle (slightly larger than grid)
    p.fillRect(
        QRectF(-0.6 * cell, -0.6 * cell, 10.2 * cell, 11.2 * cell),
        QBrush(QColor(BOARD_BG)),
    )
    ```

    **Why this works:** `QPainter.fillRect(QRectF, QBrush)` is a valid PyQt6 overload that explicitly accepts floating-point rectangle coordinates.

    **Additional change:** Add `QRectF` to the imports from `PyQt6.QtCore` at line 16 if not already present (it is already imported).

    This is the ONLY code change needed. All other `drawLine()` and `drawText()` calls already use QPointF/QLineF implicitly or accept float arguments.
