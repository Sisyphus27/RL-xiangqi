Fix the horizontal grid line loop in src/xiangqi/ui/board.py at lines 146-150.

**Root Cause Analysis:**
The current code skips drawing the horizontal line at row 4 with `if row == 4: continue`. This is incorrect. In Chinese chess:
- The river is the SPACE between rows 4 and 5 (0-indexed)
- The horizontal line at row 4 (y = 4.6*cell) is the line that row 4 pieces sit ON
- The horizontal line at row 5 (y = 5.6*cell) is the line that row 5 pieces sit ON
- Both lines should be drawn
- The "river gap" means VERTICAL lines are broken between rows 4 and 5, not horizontal lines

**Current incorrect code (lines 146-150):**
```python
        for row in range(ROWS):
            if row == 4:  # no line between river rows
                continue
            y = (row + 0.6) * cell
            p.drawLine(QLineF(0.6 * cell, y, 9.6 * cell, y))
```

**Corrected code:**
```python
        for row in range(ROWS):
            y = (row + 0.6) * cell
            p.drawLine(QLineF(0.6 * cell, y, 9.6 * cell, y))
```

Simply remove the `if row == 4: continue` conditional - all 10 horizontal lines should be drawn.

**Why this fixes the bug:**
- Before: 9 horizontal lines (missing row 4 at y = 4.6*cell)
- After: 10 horizontal lines (complete grid)
- Black's pawns at row 3 will now have the line at row 4 visible in front of them
