## Phase 07 New UI Elements

### 1. Status Bar Turn Indicator (UI-06, AI-04)

**Location:** `QMainWindow.statusBar()` (D-01)

| State | Text | Style |
|-------|------|-------|
| Red turn | "红方回合" | Default status bar text, no special styling |
| Black turn (AI thinking) | "AI 思考中..." | Default status bar text, no special styling |
| Game over | (Cleared, popup shown) | — |

**Implementation notes:**
- Use `statusBar().showMessage(text)` API
- No color/weight changes — keep Qt default status bar appearance
- Text changes immediately on turn switch
- "AI 思考中..." appears when AI thread starts, disappears when move is applied

### 2. Game Over Modal (UI-07)

**Component:** `QMessageBox.information()` modal dialog (D-03)

| Result | Dialog Title | Dialog Text |
|--------|--------------|-------------|
| RED_WINS | "对局结束" | "红胜" |
| BLACK_WINS | "对局结束" | "黑胜" |
| DRAW | "对局结束" | "和棋" |

**Button:** Single "OK" button (Qt default)

**Behavior:**
- Modal dialog blocks UI until dismissed
- After dismissal: board remains in final state
- User must manually start new game (Phase 08)

## Interaction States

### Board Interaction Control

| State | Board Interactive | Status Bar |
|-------|-------------------|------------|
| Red turn (human) | True | "红方回合" |
| Black turn (AI computing) | False | "AI 思考中..." |
| Game over | False | (cleared) |

**API:** `QXiangqiBoard.set_interactive(False)` disables all mouse clicks (D-18)

### State Flow (from CONTEXT.md D-13 to D-16)

```
User move -> result() check -> turn switch -> status update -> AI start (if black)
    |                                                              |
    v                                                              v
Board disabled                                            "AI 思考中..." shown
    |                                                              |
    +<------------------ AI move ready <---------------------------+
    |
    v
is_legal() guard -> apply move -> result() check -> turn switch -> status update -> board enabled
    |
    +--> Game over? -> show modal -> board stays in final state
```

## Acceptance Criteria (from ROADMAP.md)

1. AIPlayer ABC defines `suggest_move(snapshot: EngineSnapshot) -> Move | None`
2. EngineSnapshot dataclass encapsulates board state, turn, legal moves (thread-safe)
3. RandomAI implements and is called on black turn, returns random legal move
4. Black turn shows "AI 思考中..." in status bar, disappears after move
5. UI shows current turn indicator (red/black)
6. Game over shows modal with result (red win/black win/draw)
7. Board interaction disabled during black turn (`set_interactive(False)`)
